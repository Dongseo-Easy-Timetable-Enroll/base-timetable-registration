### 어제까지 무엇을 했는가?
    3장 유스케이스 다이어그램
    포함관계 : 유스케이스 수행 할 때 같은 기능이 있는 다른 유스케이스가 반드시 수행되는 관계
    확장관계 : 여러 유스케이스에 걸쳐 중복 사용되지 않고 특정 조건에서 한 유스케이스로만 확장되는 것

    ![image](https://github.com/Dongseo-Easy-Timetable-Enroll/base-timetable-registration/assets/167411010/924a126b-cb73-45a6-ae73-adf02e49193b)
    유스케이스 다이어그램으로 나타낸 투자거래 시스템

    5장 순차 다이어그램
    객체 간의 동적 상호작용을 시간 개념을 중심으로 모델링하는 것

    객체 : 순차 다이어그램의 맨 위에 위치하며 왼쪽에서 오른쪽으로 배열
    메시지 : 한 객체의 생명선에서 다른 객체의 생명선으로 이동하는 것을 의미
    시간 : 순차 다이어그램에서 수직방향으로 나타냄. 순서는 맨 위에서 아래로

    ![image](https://github.com/Dongseo-Easy-Timetable-Enroll/base-timetable-registration/assets/167411010/2660f649-a1b8-4d1b-b191-c7a50b822a48)
    순차 다이어그램으로 나타낸 대여시스템


    4장 클래스 다이어그램
    클래스 구성 요소 : 클래스 이름, 속성, 메서드

    ![image](https://github.com/Dongseo-Easy-Timetable-Enroll/base-timetable-registration/assets/167411010/dc8e07c4-0169-4f72-8941-be7ead88b73f)
    클래스 다이어그램으로 나타낸 날씨 정보 입력 시스템


    7장 활동 다이어그램
    시작점, 활동, 종료점, 전이로 구성됨
    활동 다이어그램은 나타내기 어려운 상황을 표현할 수 있어 업무 흐름을 분석하거나 화면 흐름을 표현할 때 유용
    
    ![image](https://github.com/Dongseo-Easy-Timetable-Enroll/base-timetable-registration/assets/167411010/767e3c8c-165a-42d4-ae62-a7b6426ee0c5)
    활동 다이어그램으로 나타낸 도서 대출 시스템


    8장 상태 다이어그램

    상태 : 객체가 존재할 수 있는 조건 중 하나로 모서리가 둥근 사각형으로 표시하고 안쪽 상단에 상태 이름을 기술한다
            객체가 가질 수 있는 가능한 모든 경우가 상태로 파악되어야 한다는 점. 객체는 파악된 상태 외의 상태는 가질 수 없음
    
    이벤트와 전이 : 이벤트에 의해 전이되는 두 상태
